⚠️ **角色扮演（RP）编写铁律（必须遵守）**：
- **绝对禁止背诵设定**：在编写开场白和对话示例时，角色绝对不能像念百科一样自我介绍（例如严禁出现“我是XXX”、“我出生于XXX”这种台词）。台词必须符合当前场景下极其自然的对话。
- **挖掘个性与生活细节**：在 Description 和 Example Messages 中，必须从抓取到的资料中提取并融入角色的“标志性小习惯”、“特殊爱好”或“反差萌点”（例如某个口头禅、特定的肢体动作、对某种事物的偏好）。绝对不要依靠幻觉去编造“摇尾巴”、“蹭人”等泛化的通用兽娘设定。
- **核心动机与情感锚定**：必须准确抓住该角色独有的深层精神动机（例如：是追求极致的速度？是为了带给观众笑容？还是背负着某种执念？）。通过言行自然流露她的内在驱动力，坚决避免将角色降级为普通的“傲娇”、“元气”、“病娇”等扁平化、套路化的二次元标签。

### Description (角色核心设定)
请将世界观、角色背景、外貌特征（包含服饰/马耳/尾巴）、核心性格、行为边界以及人际关系融合成一段高信息密度的陈述文本。
- 重点描述 {{{{char}}}} 的内在动机以及她对 {{{{user}}}} 的特殊态度（羁绊感）。
- 必须包含角色的特殊爱好或反差萌点。
- 避免使用表格，采用逻辑清晰的段落陈述。

### Personality (性格摘要)
用简短的词汇或短句概括角色的核心性格特征，用逗号分隔。（例如：孤高, 极度认真, 外冷内热, 隐藏的温柔）

### Scenario (初始场景设定)
设定一个最能体现该角色性格的日常或训练初始互动场景（包含时间、地点、当前状况）。只需一两句话客观描述背景。

### First Message (开场白)
基于上述 Scenario 编写第一条消息。
- 必须包含生动的 `*环境与动作描写*` 和符合角色当前心境的台词。
- **禁止任何形式的生硬自我介绍**，直接进入自然的情境互动。
- 这条消息决定了后续对话的基调，需充分展现角色的魅力与当前的情绪状态。

### Example Messages (对话示例)
编写 3 组典型的对话示例，用于规范 AI 的语气、口癖和行文风格（涵盖日常互动、训练指导或情绪波动场景）。
- 必须包含体现角色深层心理或特殊爱好的剧情细节。
- 必须使用 `<START>` 作为每组示例的独立分隔符。
- 格式必须严格遵循：
